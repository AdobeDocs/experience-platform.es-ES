## Actualizaciones de la interfaz de usuario {#ux}

Adobe Experience Platform está publicando actualizaciones en el dominio y en la barra de encabezado para mejorar su experiencia y unificarse con otros [!DNL Experience Cloud] aplicaciones.

- Es más fácil cambiar entre las organizaciones o a otra aplicación
- Se ha mejorado la ayuda del usuario, que incluye artículos destacados y documentación pertinente al contexto en el menú Ayuda
- Posibilidad de proporcionar comentarios sobre [!DNL Experience Platform] y tickets de compatibilidad con archivos

El despliegue de la nueva experiencia es gradual. Puede ver la experiencia en [https://experience.adobe.com/platform](https://experience.adobe.com/platform).

## [!DNL Data Science Workspace] {#dsw}

[!DNL Data Science Workspace] utiliza el aprendizaje automático y la inteligencia artificial para obtener información de sus datos. Integrado en Adobe Experience Platform, [!DNL Data Science Workspace] le ayuda a hacer predicciones utilizando sus recursos de contenido y datos en todas las soluciones de Adobe. Una de las formas [!DNL Data Science Workspace] logra esto a través del uso de [!DNL JupyterLab]. [!DNL JupyterLab] es una interfaz de usuario basada en web para [[!DNL Project Jupyter]](https://jupyter.org/) y está totalmente integrado en Adobe Experience Platform. Proporciona un entorno de desarrollo interactivo con el que los científicos de datos pueden trabajar [!DNL Jupyter] portátiles, código y datos.

**Nuevas funciones**

| Función | Descripción |
|--- | ---|
| [!DNL JupyterLab Launcher] | El [!DNL JupyterLab Launcher] ahora incluye iniciadores para [!DNL Spark] 2.4 portátiles. [!DNL Spark] Los arrancadores de portátiles 2.3 ahora están marcados como obsoletos y configurados para eliminarse en una versión posterior. |
| [!DNL Spark] 2.4 | Nueva Scala ([!DNL Spark]) y las recetas PySpark ahora utilizan [!DNL Spark] 2.4. |
| Núcleos | Scala ([!DNL Spark]) los Notebooks ahora se crean a través del núcleo Scala. Los portátiles PySpark ahora se crean mediante la [!DNL Python] Núcleo. El [!DNL Spark] y el núcleo PySpark están obsoletos y configurados para ser eliminados en una versión posterior. |
| Fórmulas | Nuevo PySpark y [!DNL Spark] las recetas ahora siguen el [!DNL Docker] flujo de trabajo similar al siguiente [!DNL Python] y recetas R. |

Para obtener más información sobre cómo migrar los blocs de notas y las fórmulas para utilizar [!DNL Spark] 2.4, consulte la [guía de acceso a datos de portátiles](../../data-science-workspace/jupyterlab/access-notebook-data.md). Para obtener información más general sobre Data Science Workspace, consulte la [documentación general](../../data-science-workspace/home.md).

## [!DNL Destinations] {#destinations}

Entrada [Real-time Customer Data Platform](../../rtcdp/overview.md), los destinos son integraciones prediseñadas con plataformas de destino que activan los datos a dichos socios de forma fluida.

**[!DNL Facebook]**

Real-Time CDP ahora admite la activación de datos en [!DNL Facebook], lo que le permite activar perfiles para su [!DNL Facebook] campañas de segmentación, personalización y supresión de audiencias basadas en correos electrónicos con hash.

Para obtener más información sobre la nueva funcionalidad, consulte la [Facebook destination](../../destinations/catalog/social/facebook.md) página.

## [!DNL Experience Platform Web SDK] y [!DNL Experience Platform Edge Network] {#edge}

El [!DNL Experience Platform Web SDK] y [!DNL Experience Platform Edge Network] permite a los usuarios enviar datos a Adobe Experience Platform y otras soluciones de Adobe en tiempo real para dispositivos y exploradores de los usuarios finales. La lista de casos de uso más reciente se encuentra en nuestra [hoja de ruta pública](https://github.com/adobe/alloy/projects/5) que se actualiza con frecuencia.

**Nuevas funciones**

| Función | Descripción |
|--- | ---|
| Compatibilidad con ECID | El SDK admite ECID de forma predeterminada sin necesidad de instalar bibliotecas ni información adicionales. |
| IU de configuración | Administre la configuración del ID con la nueva IU de configuración de Edge en [!DNL Launch], debe estar en la lista de permitidos para acceder a. |
| grupo de campos de esquema del SDK web de Adobe Experience Platform | Un grupo de campos para su uso con [!DNL Experience Platform web SDK] que abarca todos los campos compatibles. |
| Controles de consentimiento del curso | Proporciona a las empresas controles sobre la inclusión y la exclusión de [!DNL Experience Platform Web SDK]. |
| Compatibilidad con la depuración del lado del cliente en el nuevo [!DNL Experience Cloud Debugger Extension] | Consulte las solicitudes de [!DNL Experience Platform web SDK] así como los seguimientos de Edge para ver cómo fluyen los datos a través del sistema. |
| Adobe Analytics | Envío de datos a [!DNL Analytics] grupos de informes a través de la configuración de edge. XDM se aplana en los datos de contexto y admite el etiquetado de grupos múltiples. |
| Adobe Target | Compatibilidad con Adobe Target. Incluyendo VEC, Compositor basado en formularios, A/B, XT, Automated Personalization, MVT. |
| Asistencia de Adobe Audience Manager | Compatibilidad con [!DNL Audience Manager] Sincronizaciones de ID, destinos de URL y destinos de cookies. |

| Generador de objetos XDM | En la extensión de Launch, ahora puede crear objetos XDM como elementos de datos. |

Para obtener más información sobre [!DNL Platform Web SDK] y [!DNL Edge Network], consulte la [documentación](../../edge/home.md).

## [!DNL Real-Time Customer Profile] {#profile}

Adobe Experience Platform le permite impulsar experiencias coordinadas, coherentes y relevantes para sus clientes, independientemente de dónde o cuándo interactúen con su marca. Con [!DNL Real-Time Customer Profile], puede ver una vista integral de cada cliente individual que combina datos de varios canales, incluidos datos en línea, sin conexión, CRM y de terceros. [!DNL Profile] le permite consolidar los distintos datos de clientes en una vista unificada, lo que ofrece una cuenta procesable con marca de tiempo de cada interacción de clientes.

**Nuevas funciones**

| Función | Descripción |
| -----------| ---------- |
| Nuevas métricas de exportación de perfiles | Se han añadido métricas a los trabajos de exportación de perfiles que muestran el número total de perfiles exportados y el número de perfiles en cada área de nombres. |
| Nuevas métricas de Observability Insights | La API de Observability Insights ahora tiene las siguientes métricas para la transmisión de la ingesta a [!DNL Profile]: Tasa De Solicitudes Entrantes, Tasa De Ingesta Correcta, Tamaño De Registro Ingerido. |
| Extremo de GET masiva | Se ha agregado un extremo de GET masiva a [!DNL Real-Time Customer Profile] API para permitir la recuperación de varios resultados en una sola llamada de API. Ahora puede realizar GET masivos de hasta 100 ID para definiciones de segmentos, trabajos de segmentos y políticas de combinación. |
| Examinar perfil por identidad | En el [!DNL Platform] IU: ahora puede seleccionar un área de nombres de identidad y proporcionar un valor de identidad para examinar un perfil. |

## Fuentes {#sources}

Adobe Experience Platform puede introducir datos de fuentes externas, al tiempo que le permite estructurar, etiquetar y mejorar esos datos mediante [!DNL Platform] servicios. Puede introducir datos de una variedad de fuentes, como aplicaciones de Adobe, almacenamiento basado en la nube, software de terceros y su sistema CRM.

[!DNL Experience Platform] proporciona una API RESTful y una interfaz de usuario interactiva que le permite configurar conexiones de origen para varios proveedores de datos con facilidad. Estas conexiones de origen le permiten autenticarse y conectarse a sistemas de almacenamiento externos y servicios CRM, establecer tiempos para ejecuciones de ingesta y administrar el rendimiento de ingesta de datos.

**Nuevas funciones**

| Función | Descripción |
| ------- | ----------- |
| Compatibilidad adicional con API e IU para sistemas de almacenamiento en la nube | Nuevos conectores de origen para [!DNL Azure File Storage]. |
| Compatibilidad adicional con API e IU para bases de datos | Nuevos conectores de origen para [!DNL Azure Data Explorer], IBM DB2 y [!DNL Oracle DB]. |
| ADOBE AUDIENCE MANAGER a [!DNL Experience Platform] compartir datos | El proceso de aprovisionamiento para [!DNL Audience Manager] se ha actualizado el conector. [!DNL Audience Manager] conjuntos de datos para [!DNL Real-Time Customer Profile] ahora están desactivadas de forma predeterminada. Puede elegir manualmente a qué conjuntos de datos promocionar [!DNL Profile]. La nueva configuración predeterminada no es retroactiva y solo afecta al aprovisionamiento de las nuevas [!DNL Audience Manager] conectores. Consulte más información en la [Guía del usuario de conjuntos de datos](../../catalog/datasets/user-guide.md). |
