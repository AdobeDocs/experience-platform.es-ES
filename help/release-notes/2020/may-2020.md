## Actualizaciones de la interfaz de usuario {#ux}

Adobe Experience Platform is releasing updates to the domain and header bar to improve your experience and unify with other [!DNL Experience Cloud] applications.

- Es más fácil cambiar entre las organizaciones o a otra aplicación
- Se ha mejorado la ayuda del usuario, que incluye artículos destacados y documentación pertinente al contexto en el menú Ayuda
- Ability to give feedback about the [!DNL Experience Platform] and file support tickets

El desarrollo de la nueva experiencia es gradual. Puede realizar la vista de la experiencia en [https://experience.adobe.com/platform](https://experience.adobe.com/platform).

## [!DNL Data Science Workspace] {#dsw}

[!DNL Data Science Workspace] utiliza el aprendizaje automático y la inteligencia artificial para generar perspectivas a partir de los datos. Integrado en Adobe Experience Platform, [!DNL Data Science Workspace] le ayuda a realizar predicciones mediante el uso de contenido y recursos de datos en las soluciones de Adobe. Una de las formas de [!DNL Data Science Workspace] lograrlo es mediante el uso de [!DNL JupyterLab]. [!DNL JupyterLab] es una interfaz de usuario basada en web para [[!DNL Project Jupyter]](https://jupyter.org/) y está estrechamente integrada en Adobe Experience Platform. Proporciona un entorno de desarrollo interactivo para que los científicos de datos trabajen con [!DNL Jupyter] portátiles, código y datos.

**Nuevas funciones**

| Función | Descripción |
|--- | ---|
| [!DNL JupyterLab Launcher] | El [!DNL JupyterLab Launcher] ahora incluye los inicios de [!DNL Spark] portátiles 2.4. [!DNL Spark] Los primeros del bloc de notas 2.3 ahora están marcados como obsoletos y configurados para eliminarse en una versión posterior. |
| [!DNL Spark] 2.4 | Las nuevas fórmulas Scala ([!DNL Spark]) y PySpark ahora utilizan [!DNL Spark] 2.4. |
| Kernels | Los portátiles Scala ([!DNL Spark]) ahora se crean a través del núcleo Scala. Los blocs de notas PySpark ahora se crean mediante el [!DNL Python] kernel. El núcleo [!DNL Spark] y PySpark están obsoletos y se han configurado para eliminarse en una versión posterior. |
| Fórmulas | Las nuevas fórmulas y PySpark ahora siguen el [!DNL Spark] flujo de trabajo de forma similar a las fórmulas [!DNL Docker] [!DNL Python] y R. |

Para obtener más información sobre la migración de los blocs de notas y las fórmulas para utilizar [!DNL Spark] 2.4, consulte la guía [de acceso a datos del](../../data-science-workspace/jupyterlab/access-notebook-data.md)bloc de notas. Para obtener más información general sobre el espacio de trabajo de ciencia de datos, consulte la documentación [](../../data-science-workspace/home.md)general.

## [!DNL Destinations] {#destinations}

En la plataforma [de datos del cliente en tiempo](../../rtcdp/overview.md)real, los destinos son integraciones prediseñadas con plataformas de destino que activan los datos a dichos socios de una manera transparente.

**[!DNL Facebook]**

Ahora, CDP en tiempo real admite la activación de datos a [!DNL Facebook], lo que le permite activar perfiles para sus [!DNL Facebook] campañas de objetivo de audiencias, personalización y supresión en base a correos electrónicos con hash.

Para obtener más información sobre la nueva funcionalidad, consulte la página de destino [de](../../destinations/catalog/social/facebook.md) Facebook.

La lista más reciente de casos de uso puede encontrarse en nuestra hoja de ruta [pública](https://github.com/adobe/alloy/projects/5) , que se actualiza con frecuencia.

**Nuevas funciones**

| Función | Descripción |
|--- | ---|
| Compatibilidad con ECID | El SDK admite ECID de forma predeterminada sin tener que instalar ninguna biblioteca o información adicional. |
| UI de configuración | Administre la configuración de ID con la nueva interfaz de usuario de configuración de Edge en [!DNL Launch], debe estar en la lista de permitidos para acceder a ella. |
| Adobe Experience Platform Web SDK Mixin | Mezcla para usar con el [!DNL Experience Platform web SDK] que abarca todos los campos admitidos. |
| Controles de consentimiento del curso | Proporciona controles de compañías sobre la opción de inclusión y exclusión del [!DNL Experience Platform Web SDK]. |
| Compatibilidad de depuración del lado del cliente en el nuevo [!DNL Experience Cloud Debugger Extension] | Consulte las solicitudes desde el sistema, así como los seguimientos de borde, para ver cómo fluyen los datos a través del sistema. [!DNL Experience Platform web SDK] |
| Adobe Analytics | Envíe datos a los grupos [!DNL Analytics] de informes mediante la configuración de Edge. XDM se acopla en datos de contexto, admite el etiquetado de grupos múltiples. |
| Adobe Target | Compatibilidad con Adobe Target. Incluido VEC, Compositor basado en formularios, A/B, XT, Automated Personalization, MVT. |

## [!DNL Real-time Customer Profile] {#profile}

Adobe Experience Platform le permite dirigir experiencias coordinadas, coherentes y relevantes para sus clientes, independientemente de dónde o cuándo interactúen con su marca. Con [!DNL Real-time Customer Profile], puede ver una vista holística de cada cliente individual que combina datos de varios canales, incluidos datos en línea, sin conexión, CRM y de terceros. [!DNL Profile] le permite consolidar sus datos dispares de clientes en una vista unificada que ofrece una cuenta procesable con marca de hora de cada interacción con los clientes.

**Nuevas funciones**

| Función | Descripción |
| -----------| ---------- |
| Nuevas métricas de exportación de Perfil | Métricas añadidas para los trabajos de exportación de perfiles que muestran la cantidad total de perfiles exportados y la cantidad de perfiles en cada Área de nombres. |
| Nuevas métricas de perspectivas de visibilidad | La API de perspectivas de capacidad de observación ahora tiene las siguientes métricas para transmitir la ingesta a [!DNL Profile]: Tasa De Solicitud Entrante, Tasa De Ingesta Correcta, Tamaño De Registro Ingerido. |
| Extremo de GET masivo | Se añadió el extremo de GET masiva a la API para permitir la recuperación de varios resultados en una sola llamada de API. [!DNL Real-time Customer Profile] Ahora puede realizar GET masivas de hasta 100 ID para definiciones de segmentos, trabajos de segmentos y políticas de combinación. |
| Examinar perfil por identidad | En la interfaz de usuario, ahora puede seleccionar una Área de nombres de identidad y proporcionar un valor de identidad para examinar un perfil. [!DNL Platform] |

## Fuentes {#sources}

Adobe Experience Platform puede ingerir datos de fuentes externas y, al mismo tiempo, puede estructurarlos, etiquetarlos y mejorarlos mediante [!DNL Platform] servicios. Puede ingestar datos de una variedad de fuentes, como aplicaciones de Adobe, almacenamiento basado en la nube, software de terceros y su sistema CRM.

[!DNL Experience Platform] proporciona una API RESTful y una interfaz de usuario interactiva que le permite configurar fácilmente las conexiones de origen para varios proveedores de datos. Estas conexiones de origen le permiten autenticarse y conectarse a sistemas de almacenamiento externos y servicios CRM, establecer tiempos para ejecuciones de ingestión y administrar el rendimiento de la ingesta de datos.

**Nuevas funciones**

| Función | Descripción |
| ------- | ----------- |
| Compatibilidad adicional de API e interfaz de usuario para sistemas de almacenamiento en la nube | Conectores de origen nuevos para [!DNL Azure File Storage]. |
| Compatibilidad adicional de API e interfaz de usuario con bases de datos | Nuevos conectores de origen para [!DNL Azure Data Explorer], IBM DB2 y [!DNL Oracle DB]. |
| Adobe Audience Manager para compartir [!DNL Experience Platform] datos | Se ha actualizado el proceso de aprovisionamiento del [!DNL Audience Manager] conector. [!DNL Audience Manager] los conjuntos de datos para [!DNL Real-time Customer Profile] ahora están deshabilitados de forma predeterminada. Puede elegir manualmente los conjuntos de datos a los que desea promocionar [!DNL Profile]. La nueva configuración predeterminada no es retroactiva y solo afecta al aprovisionamiento de nuevos [!DNL Audience Manager] conectores. Consulte más información en la guía [del usuario de](../../catalog/datasets/user-guide.md)conjuntos de datos. |
