## Actualizaciones de la interfaz de usuario {#ux}

Adobe Experience Platform está publicando actualizaciones en el dominio y en la barra de encabezado para mejorar su experiencia y unificarse con otras aplicaciones [!DNL Experience Cloud].

- Es más fácil cambiar entre las organizaciones o a otra aplicación
- Se ha mejorado la ayuda del usuario, que incluye artículos destacados y documentación pertinente al contexto en el menú Ayuda
- Posibilidad de proporcionar comentarios sobre los [!DNL Experience Platform] y los tickets de compatibilidad con archivos

El despliegue de la nueva experiencia es gradual. Puede ver la experiencia en [https://experience.adobe.com/platform](https://experience.adobe.com/platform).

## [!DNL Data Science Workspace] {#dsw}

[!DNL Data Science Workspace] utiliza el aprendizaje automático y la inteligencia artificial para extraer perspectivas de sus datos. Integrado en Adobe Experience Platform, [!DNL Data Science Workspace] le ayuda a hacer predicciones utilizando sus recursos de contenido y datos en las soluciones de Adobe. Una de las maneras en que [!DNL Data Science Workspace] lo logra es a través del uso de [!DNL JupyterLab]. [!DNL JupyterLab] es una interfaz de usuario basada en web para  [[!DNL Project Jupyter]](https://jupyter.org/) y está estrechamente integrada en Adobe Experience Platform. Proporciona un entorno de desarrollo interactivo para que los científicos de datos trabajen con [!DNL Jupyter] portátiles, código y datos.

**Nuevas funciones**

| Función | Descripción |
|--- | ---|
| [!DNL JupyterLab Launcher] | El [!DNL JupyterLab Launcher] ahora incluye el inicio de los [!DNL Spark] portátiles 2.4. [!DNL Spark] Los inicios de bloc de notas 2.3 ahora están marcados como obsoletos y configurados para eliminarse en una versión posterior. |
| [!DNL Spark] 2,4 | Las nuevas fórmulas Scala ([!DNL Spark]) y PySpark ahora utilizan [!DNL Spark] 2.4. |
| Kernels | Los portátiles Scala ([!DNL Spark]) ahora se crean a través del núcleo Scala. Los blocs de notas PySpark ahora se crean mediante el kernel [!DNL Python]. El kernel [!DNL Spark] y PySpark están obsoletos y configurados para ser eliminados en una versión posterior. |
| Fórmulas | Las nuevas fórmulas PySpark y [!DNL Spark] ahora siguen el flujo de trabajo [!DNL Docker] similar a las fórmulas [!DNL Python] y R. |

Para obtener más información sobre la migración de los blocs de notas y las fórmulas para utilizar [!DNL Spark] 2.4, consulte la [guía de acceso a datos del bloc de notas](../../data-science-workspace/jupyterlab/access-notebook-data.md). Para obtener información más general sobre Data Science Workspace, consulte la [documentación general](../../data-science-workspace/home.md).

## [!DNL Destinations] {#destinations}

En la [Plataforma de datos del cliente en tiempo real](../../rtcdp/overview.md), los destinos son integraciones prediseñadas con plataformas de destino que activan los datos a esos socios de una manera transparente.

**[!DNL Facebook]**

CDP en tiempo real ahora admite la activación de datos en [!DNL Facebook], lo que le permite activar perfiles para sus campañas [!DNL Facebook] para segmentación, personalización y supresión de audiencias en función de correos electrónicos con hash.

Para obtener más información sobre la nueva funcionalidad, consulte la página [Destino de Facebook](../../destinations/catalog/social/facebook.md).

## [!DNL Experience Platform Web SDK] y [!DNL Experience Platform Edge Network] {#edge}

Los [!DNL Experience Platform Web SDK] y [!DNL Experience Platform Edge Network] permiten a los usuarios enviar datos a Adobe Experience Platform y otras soluciones de Adobe en tiempo real para dispositivos y navegadores de usuario final. La lista más reciente de casos de uso se encuentra en nuestra [hoja de ruta pública](https://github.com/adobe/alloy/projects/5) que se actualiza con frecuencia.

**Nuevas funciones**

| Función | Descripción |
|--- | ---|
| Compatibilidad con ECID | El SDK admite ECID de forma predeterminada sin tener que instalar bibliotecas ni información adicional. |
| Interfaz de usuario de configuración | Administre la configuración de ID con la nueva interfaz de usuario de configuración de Edge en [!DNL Launch], debe estar en la lista de permitidos para acceder. |
| Grupo de campos de esquema del SDK web de Adobe Experience Platform | Un grupo de campos para usar con [!DNL Experience Platform web SDK] que abarca todos los campos admitidos. |
| Controles de consentimiento del curso | Proporciona a las empresas controles sobre la inclusión y la exclusión del [!DNL Experience Platform Web SDK]. |
| Compatibilidad de depuración del lado del cliente en el nuevo [!DNL Experience Cloud Debugger Extension] | Consulte las solicitudes de [!DNL Experience Platform web SDK] , así como los seguimientos de borde, para ver cómo fluyen los datos a través del sistema. |
| Adobe Analytics | Envíe datos a grupos de informes [!DNL Analytics] mediante la configuración perimetral. XDM se acopla en datos de contexto y admite el etiquetado de grupos múltiples. |
| Adobe Target | Compatibilidad con Adobe Target. Incluido VEC, Compositor basado en formularios, A/B, XT, Automated Personalization, MVT. |
| Asistencia de Adobe Audience Manager | Compatibilidad con sincronizaciones de ID [!DNL Audience Manager], destinos de URL y destinos de cookies. |
| Sincronización de identidad | Se cambió el nombre `setCustomersIds` a `syncIdentity` para que sea más claro. |
| Generador de objetos XDM | En la extensión de Launch, ahora puede crear objetos XDM como elementos de datos. |

Para obtener más información sobre [!DNL Platform Web SDK] y [!DNL Edge Network], consulte la [documentación](../../edge/home.md).

## [!DNL Real-time Customer Profile] {#profile}

Adobe Experience Platform le permite ofrecer experiencias coordinadas, coherentes y relevantes a sus clientes, independientemente de dónde o cuándo interactúen con su marca. Con [!DNL Real-time Customer Profile], puede ver una vista integral de cada cliente individual que combina datos de varios canales, incluidos datos en línea, sin conexión, CRM y de terceros. [!DNL Profile] le permite consolidar sus diferentes datos de clientes en una vista unificada, que ofrece una cuenta procesable con marca de tiempo de cada interacción con los clientes.

**Nuevas funciones**

| Función | Descripción |
| -----------| ---------- |
| Nuevas métricas de exportación de perfil | Se han agregado métricas a los trabajos de exportación de perfiles que muestran el número total de perfiles exportados y el número de perfiles en cada área de nombres. |
| Nuevas métricas de Perspectivas de la capacidad de observación | La API de Observability Insights ahora tiene las siguientes métricas para la transmisión de ingesta a [!DNL Profile]: Tasa De Solicitud Entrante, Tasa De Ingesta Correcta, Tamaño De Registro Ingestado. |
| Punto final de GET masivo | Se ha agregado un extremo de GET masivo a la API [!DNL Real-time Customer Profile] para permitir la recuperación de varios resultados en una sola llamada de API. Ahora puede realizar GET masivas de hasta 100 ID para definiciones de segmentos, trabajos de segmentos y políticas de combinación. |
| Examinar perfil por identidad | En la interfaz de usuario de [!DNL Platform] ahora puede seleccionar un área de nombres de identidad y proporcionar un valor de identidad para examinar un perfil. |

## Fuentes {#sources}

Adobe Experience Platform puede ingerir datos de orígenes externos, al mismo tiempo que le permite estructurarlos, etiquetarlos y mejorarlos mediante los servicios [!DNL Platform]. Puede ingerir datos de una variedad de fuentes, como aplicaciones de Adobe, almacenamiento basado en la nube, software de terceros y su sistema CRM.

[!DNL Experience Platform] proporciona una API de RESTful y una interfaz de usuario interactiva que le permite configurar conexiones de origen para varios proveedores de datos con facilidad. Estas conexiones de origen le permiten autenticarse y conectarse a sistemas de almacenamiento externos y servicios CRM, establecer tiempos para ejecutar la ingesta y administrar el rendimiento de ingesta de datos.

**Nuevas funciones**

| Función | Descripción |
| ------- | ----------- |
| Compatibilidad adicional de API e IU con los sistemas de almacenamiento en la nube | Nuevos conectores de origen para [!DNL Azure File Storage]. |
| Compatibilidad adicional de API e IU con bases de datos | Nuevos conectores de origen para [!DNL Azure Data Explorer], IBM DB2 y [!DNL Oracle DB]. |
| Uso compartido de datos de Adobe Audience Manager con [!DNL Experience Platform] | Se ha actualizado el proceso de aprovisionamiento del conector [!DNL Audience Manager]. [!DNL Audience Manager] los conjuntos de datos de ahora  [!DNL Real-time Customer Profile] están deshabilitados de forma predeterminada. Puede elegir manualmente qué conjuntos de datos promocionar a [!DNL Profile]. La nueva configuración predeterminada no es retroactiva y solo afecta al aprovisionamiento de nuevos conectores [!DNL Audience Manager]. Consulte más información en la [Guía del usuario de conjuntos de datos](../../catalog/datasets/user-guide.md). |
