## Ingesta de datos

Adobe Experience Platform proporciona un completo conjunto de funciones para ingestar cualquier tipo y latencia de datos. Puede realizar la ingesta mediante las API de flujo continuo o por lotes mediante conectores creados por Adobe, socios de integración de datos o la interfaz de usuario de la plataforma Adobe Experience Platform.

Para obtener documentación sobre el producto, consulte la información general [sobre la ingestión de](../../ingestion/home.md)datos.

### Ingesta por lotes

Utilice la API de inserción por lotes para transferir cualquier tipo de datos a Adobe Experience Platform. La API de ingestión masiva proporciona un mecanismo seguro y escalable para la ingesta de datos que se ajustan a esquemas XDM o no XDM.

| Función | Descripción |
| -----------| ---------- |
| Compatibilidad con formato enriquecido | Admite ingestión de JSON, Parquet y CSV |
| Validación de datos | Se realizan tres niveles de validación en los datos: Esquemas, tipo de datos y restricciones |
| Cifrado de datos en movimiento | Protege y asegura los datos mientras se transmiten a través de las redes de datos. |
| Cargas de varias partes | Cargas de archivos grandes admitidas para archivos de 256 MB o más |
| Particionado automáticamente | Los datos se dividen automáticamente para mejorar la escalabilidad y optimizar el rendimiento de los grandes conjuntos de datos. |

**Problemas conocidos**

None.

Para obtener documentación del producto, consulte la información general [sobre la ingestión de](../../ingestion/batch-ingestion/overview.md)lotes.

### Conectores

Adobe Experience Platform Connectors le ayuda a ingestar fácilmente sus datos de varias fuentes, lo que le permite estructurar, etiquetar y mejorar sus datos mediante los servicios de Experience Platform. Los conectores de plataforma le permiten autenticarse en sistemas de almacenamiento basados en la nube y en servicios CRM, establecer tiempos para ejecuciones de ingestión y administrar el rendimiento de la ingesta de datos.

| Función | Descripción |
| -----------| ---------- |
| Conectores de Almacenamiento basados en la nube | Ingresar datos desde el almacenamiento en la nube Microsoft Azure Blob y Amazon S3 a Adobe Experience Platform |
| Microsoft Dynamics CRM y Salesforce CRM | Descubrir todos los datos disponibles en Microsoft Dynamics o Salesforce CRM. Ingrese cualquier objeto, estándar o personalizado desde Microsoft Dynamics o Salesforce CRM. Los datos incrementales se pueden ingerir por hora o por día. |  |

**Problemas conocidos**

* Los conectores de Almacenamiento basados en la nube solo se pueden usar mediante llamadas de API.


Para obtener documentación sobre el producto, consulte Descripción general [de](../../sources/home.md)los conectores.

### Transmisión por flujo continuo

La inserción de flujo continuo envía datos a la plataforma de experiencia en tiempo real desde dispositivos cliente y servidor. Se pueden realizar actualizaciones en Perfiles de clientes en tiempo real con latencia de subsegundos, con compatibilidad con la recopilación de datos autenticados y las cargas de varios registros.

| Función | Descripción |
| -----------| ---------- |
| Validación sincrónica | Permite el descubrimiento inmediato de errores en la carga útil de JSON, lo que mejora la velocidad del desarrollador. |
| Monitoreo end-to-end | Supervisa los datos de flujo continuo desde la ingesta hasta el consumo en el lago de datos y el Perfil del cliente en tiempo real. |
| Recopilación de datos autenticados | Activa o desactiva la opción para requerir tokens de autenticación de clientes. Permite a los servicios de Adobe Experience Platform diferenciar entre registros procedentes de fuentes de confianza y fuentes de confianza. |
| Enviar datos en tiempo real mediante Launch | Utiliza la extensión Adobe Experience Platform en Launch para enviar fácilmente datos con formato XDM desde dispositivos cliente en tiempo real. |

**Problemas conocidos**

None.

Para obtener documentación sobre el producto, consulte [Introducción](../../ingestion/streaming-ingestion/overview.md)a la ingesta de flujo continuo.

## Modelo de datos de experiencia (XDM) y sistema XDM

La estandarización y la interoperabilidad son conceptos clave que sustentan la plataforma de experiencia. El modelo de datos de experiencia (XDM), impulsado por Adobe, es un esfuerzo para estandarizar los datos de experiencia del cliente y definir esquemas para la administración de la experiencia del cliente.

XDM es una especificación públicamente documentada diseñada para mejorar el poder de las experiencias digitales. Proporciona estructuras y definiciones comunes para que cualquier aplicación se comunique con los servicios de la plataforma Adobe Experience. Al cumplir con los estándares XDM, todos los datos de experiencia del cliente se pueden incorporar a una representación común que ofrece perspectivas de una manera más rápida e integrada. Puede obtener perspectivas valiosas de las acciones de los clientes, definir audiencias de clientes a través de segmentos y utilizar atributos de clientes para fines de personalización.

XDM es el mecanismo que permite a Experience Cloud, con la tecnología de Adobe Experience Platform, entregar el mensaje correcto a la persona adecuada, en el canal correcto y en el momento preciso.

La metodología en la que se basa la plataforma de experiencia, el sistema XDM hace operativos los esquemas del modelo de datos de experiencia para su uso por los componentes de la plataforma de experiencia.

| Función | Descripción |
| -----------| ---------- |
| Registro de Esquemas | El Registro de Esquemas proporciona una interfaz de usuario (Editor de Esquemas) y una API RESTful desde la que se pueden ver y administrar todos los recursos relacionados con el esquema. El Editor de Esquemas y la API del Registro de Esquemas también se pueden utilizar para crear y administrar nuevos esquemas y recursos que son exclusivos de cada organización de IMS. |
| Biblioteca de Esquemas | La biblioteca de Esquemas contiene todos los recursos del Registro de Esquemas, incluidos los definidos por una organización de IMS individual y los que ponen a disposición de todos los usuarios Adobe, los socios de la plataforma de experiencia y los proveedores cuyas aplicaciones están integradas con la plataforma de experiencia. |

**Problemas conocidos**

None.

Para obtener más información sobre cómo trabajar con XDM mediante la API del Registro de Esquema y el Editor de Esquemas, lea la documentación [del sistema](../../xdm/home.md)XDM.

### Perfil

Perfil es un almacén de entidades de búsqueda genérico que combina datos entre varios recursos de datos empresariales y proporciona acceso a esos datos en forma de perfiles de consumo en tiempo real y eventos de series temporales relacionados, denominados vista de unión. La vista unificada ayuda a informar y potenciar las acciones en canales, plataformas e integraciones de soluciones de Adobe. Además, el Perfil se utiliza para alimentar el aprendizaje automático y Sensei. Las API de Perfil también pueden utilizarse directamente para enriquecer la funcionalidad de soluciones de terceros, sistemas CRM y soluciones propietarias.

| Función | Descripción |
| -----------| ---------- |
| perfil del cliente en tiempo real | Perfil ingiere datos mediante entradas de datos, los combina con datos existentes en el almacén de perfiles y crea o modifica la vista unificada en tiempo casi real. Estos datos, que evolucionan a tiempo con las experiencias de sus clientes, se pueden utilizar para realizar recomendaciones personalizadas. |
| vista de Unión | Una de las características clave de Perfil es la capacidad de unificar datos de varios canales. El Perfil puede proporcionar una vista combinada de los individuos en su base de consumidores, entre conjuntos de datos e identidades vinculadas, conocida como la vista de unión. |
| Edge | Edge Service sirve como marco para la recopilación de datos de baja latencia, la informática conectable y la rápida activación de datos en todos los canales direccionables. Edge Service facilita la personalización en tiempo real. |

**Problemas conocidos**

None.

Para obtener documentación sobre el producto, consulte Información general sobre [Perfiles](../../profile/home.md).

### Servicio de identidad

El servicio de identidad resuelve el desafío fundamental que plantean las identidades desconectadas de sus clientes, debido a la naturaleza fragmentada de sus datos en cada uno de los sistemas dispares en juego en el ecosistema de su organización. Esto se logra mediante el enlace de identidades entre dispositivos y entre los distintos sistemas utilizados por los consumidores para interactuar con su marca. De este modo, el servicio de identidad facilita una comprensión completa de sus clientes y de su comportamiento para que pueda ofrecer experiencias digitales impactantes en tiempo real.

| Función | Descripción |
| -----------| ---------- |
| Gráfico de identidad | Un gráfico de identidad es un mapa de las relaciones entre identidades que se actualiza con la actividad del cliente en tiempo casi real. Identity Service administra un gráfico de identidad visible únicamente por su organización y creado en función de sus datos, denominado gráfico privado. El servicio de identidad aumenta el gráfico privado cuando un registro de datos ingestados contiene más de una identidad, agregando relaciones entre estas identidades. |

**Problemas conocidos**

None.

Para obtener documentación sobre el producto, consulte [Identity Service Overview](../../identity-service/home.md).

### Servicio de segmentación

El servicio de segmentación define un subconjunto particular de perfiles de su almacén de perfiles, describiendo los criterios que distinguen a un grupo comercializable de personas de su almacén de perfiles. Los segmentos pueden basarse en datos de registros (como información demográfica) o en eventos de series temporales que representen los puntos de contacto del cliente con su marca.

Por ejemplo: en una campaña de correo electrónico llamada &quot;¿Olvidó comprar sus zapatillas?&quot;, puede que desee un segmento de audiencia de todos los usuarios que buscaron zapatos deportivos en los últimos 30 días, pero que no completaron una compra. O bien, es posible que desee destinatario el contenido web para que solo se muestre a los usuarios con una determinada clasificación en el programa de recompensas.

| Función | Descripción |
| -----------| ---------- |
| API de segmentación | Los servicios RESTful le proporcionan acceso mediante programación para generar segmentos de clientes. |
| Generador de segmentos | El generador de segmentos es un espacio de trabajo sólido e intuitivo para crear segmentos de clientes a partir de los datos de Perfil. |

**Problemas conocidos**

None.

Para obtener documentación sobre el producto, consulte la información general sobre el servicio [de segmentación](../../segmentation/home.md)

## Gobierno de datos

Adobe Experience Platform permite a las compañías reunir datos de varios sistemas empresariales para permitir que los especialistas en marketing identifiquen, entiendan y capten mejor a los clientes. La plataforma de experiencia incluye una infraestructura de administración de datos end-to-end, que incluye el etiquetado y cumplimiento del uso de datos (DULE), para garantizar el uso adecuado de los datos dentro de la plataforma y cuando se comparten entre sistemas.

La administración de datos es una serie de estrategias y tecnologías utilizadas para administrar los datos de los clientes y garantizar el cumplimiento de las regulaciones, restricciones y políticas aplicables al uso de los datos. Desempeña un papel clave dentro de la plataforma de experiencias en varios niveles, incluyendo la catalogación, el linaje de datos, el etiquetado del uso de datos, las políticas de acceso a datos y el control de acceso en los datos para las acciones de marketing.

Para comenzar con la administración de datos, es necesario comprender a fondo las regulaciones, las obligaciones contractuales y las políticas corporativas que se aplican a los datos del cliente. Desde allí, los datos pueden clasificarse aplicando las etiquetas de uso de datos correspondientes, y su uso puede controlarse mediante la definición de políticas de uso de datos.

El marco DULE simplifica y racionaliza el proceso de categorización de datos y creación de políticas de uso de datos a través de la interfaz de usuario de la plataforma de experiencia y la API del servicio de políticas DULE.

| Función | Descripción |
| -----------| ---------- |
| Etiquetar entidades de la plataforma de experiencias | Aplique y administre etiquetas de uso de datos en el nivel de campo de conexión, conjunto de datos y conjunto de datos en base a etiquetas clasificadas predefinidas. |
| Crear directivas de uso de datos | Defina y administre políticas de uso de datos en función de la presencia de etiquetas y acciones de mercadotecnia solicitadas. |
| Comprobar infracciones de directiva | Evaluar si se infringen las directivas de uso definidas al realizar acciones de marketing en los datos de la plataforma. |

**Problemas conocidos**

* None.

Para obtener más información, consulte la información general [sobre el Gobierno](../../data-governance/home.md)de datos).
