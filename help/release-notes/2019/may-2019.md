## [!DNL Data Ingestion] {#ingestion}

Adobe Experience Platform proporciona un completo conjunto de funciones para ingerir cualquier tipo y latencia de datos. Puede realizar la ingesta mediante las API de flujo continuo o por lotes mediante conectores creados por Adobe, socios de integración de datos o la interfaz de usuario de Adobe Experience Platform.

Para obtener documentación sobre el producto, consulte la información general [sobre la ingestión de](../../ingestion/home.md)datos.

### Ingesta por lotes

Utilice la API de inserción por lotes para ingestar cualquier tipo de datos en Adobe Experience Platform. La API de ingestión masiva proporciona un mecanismo seguro y escalable para la ingesta de datos que se ajustan a esquemas XDM o no XDM.

| Función | Descripción |
| -----------| ---------- |
| Compatibilidad con formato enriquecido | Admite ingestión de JSON, Parquet y CSV |
| Validación de datos | Se realizan tres niveles de validación en los datos: Esquemas, tipo de datos y restricciones |
| Cifrado de datos en movimiento | Protege y asegura los datos mientras se transmiten a través de las redes de datos. |
| Cargas de varias partes | Cargas de archivos grandes admitidas para archivos de 256 MB o más |
| Particionado automáticamente | Los datos se dividen automáticamente para mejorar la escalabilidad y optimizar el rendimiento de los grandes conjuntos de datos. |

**Problemas conocidos**

Ninguna.

Para obtener documentación del producto, consulte la información general [sobre la ingestión de](../../ingestion/batch-ingestion/overview.md)lotes.

### Conectores

Adobe Experience Platform Connectors le ayuda a ingestar fácilmente sus datos desde múltiples fuentes, permitiéndole estructurar, etiquetar y mejorar sus datos mediante [!DNL Experience Platform] servicios. [!DNL Platform] los conectores le permiten autenticarse en sistemas de almacenamiento basados en la nube y en servicios CRM, establecer tiempos para ejecuciones de ingestión y administrar el rendimiento de la ingesta de datos.

| Función | Descripción |
| -----------| ---------- |
| Conectores de Almacenamiento basados en la nube | Ingestar datos desde [!DNL Microsoft Azure Blob] y [!DNL Amazon S3] cloud almacenamiento a Adobe Experience Platform |
| [!DNL Microsoft Dynamics CRM] &amp; [!DNL Salesforce CRM] | Descubrir todos los datos disponibles en [!DNL Microsoft Dynamics] o [!DNL Salesforce] CRM. Ingrese cualquier objeto, estándar o personalizado desde [!DNL Microsoft Dynamics] o [!DNL Salesforce] CRM. Los datos incrementales se pueden ingerir por hora o por día. |  |

**Problemas conocidos**

* Los conectores de Almacenamiento basados en la nube solo se pueden usar mediante llamadas de API.


Para obtener documentación sobre el producto, consulte Descripción general [de](../../sources/home.md)los conectores.

### Transmisión por flujo continuo

La ingestión por flujo continuo envía datos a [!DNL Experience Platform] en tiempo real desde dispositivos cliente y servidor. Se pueden realizar actualizaciones en latencia de subsegundos [!DNL Real-Time Customer Profiles] con compatibilidad con la recopilación de datos autenticada y las cargas de varios registros.

| Función | Descripción |
| -----------| ---------- |

| Monitoreo end-to-end | Supervisa los datos de flujo continuo desde la ingestión hasta el consumo en el [!DNL Data Lake] y [!DNL Real-time Customer Profile]. |
| Recopilación de datos autenticados | Activa o desactiva la opción para requerir tokens de autenticación de clientes. Permite a los servicios de Adobe Experience Platform diferenciar entre registros procedentes de fuentes de confianza y de fuentes de confianza. |
| Enviar datos en tiempo real a través de [!DNL Launch] | Utiliza la extensión Adobe Experience Platform en [!DNL Launch] para enviar fácilmente datos con formato XDM desde dispositivos cliente en tiempo real. |

Ninguna.

## [!DNL Experience Data Model] (XDM) {#xdm}

La estandarización y la interoperabilidad son conceptos clave que sustentan [!DNL Experience Platform]. [!DNL Experience Data Model] (XDM), impulsado por el Adobe, es un esfuerzo para estandarizar los datos de experiencia del cliente y definir esquemas para la administración de la experiencia del cliente.

XDM es una especificación públicamente documentada diseñada para mejorar el poder de las experiencias digitales. Proporciona estructuras y definiciones comunes para que cualquier aplicación se comunique con los servicios de Adobe Experience Platform. Al cumplir con los estándares XDM, todos los datos de experiencia del cliente se pueden incorporar a una representación común que ofrece perspectivas de una manera más rápida e integrada. Puede obtener perspectivas valiosas de las acciones de los clientes, definir audiencias de clientes a través de segmentos y utilizar atributos de clientes para fines de personalización.

XDM es el mecanismo que permite a [!DNL Experience Clou]d, con la tecnología de Adobe Experience Platform, entregar el mensaje correcto a la persona correcta, en el canal correcto, en el momento preciso.

La metodología en la que [!DNL Experience Platform] se basa, [!DNL XDM System] pone en funcionamiento [!DNL Experience Data Model] esquemas para su uso por parte de [!DNL Experience Platform] los componentes.

| Función | Descripción |
| -----------| ---------- |
| [!DNL Schema Registry] | El [!DNL Schema Registry] proporciona una interfaz de usuario ([!DNL Schema Editor]) y una API RESTful desde la cual se pueden ver y administrar todos los recursos relacionados con el esquema. La [!DNL Schema Editor] API y [!DNL Schema Registry] la API también se pueden usar para crear y administrar nuevos esquemas y recursos que son exclusivos de cada organización de IMS. |
| [!DNL Schema Library] | El [!DNL Schema Library] contiene todos los [!DNL Schema Registry] recursos, incluidos los definidos por una organización de IMS individual y los puestos a disposición de todos los usuarios por Adobes, socios [!DNL Experience Platform] y proveedores cuyas aplicaciones están integradas con [!DNL Experience Platform]. |

**Problemas conocidos**

Ninguna.

Para obtener más información sobre cómo trabajar con XDM mediante la [!DNL Schema Registry] API y [!DNL Schema Editor], lea la documentación [del sistema](../../xdm/home.md)XDM.

## [!DNL Real-time Customer Profile] {#profile}

[!DNL Profile] es un almacén de entidades de búsqueda genérico que combina datos entre varios recursos de datos empresariales y proporciona acceso a esos datos en forma de perfiles de consumo en tiempo real y eventos de series temporales relacionados, denominados la vista de unión. La vista unificada ayuda a informar y potenciar las acciones en integraciones de soluciones de canales [!DNL Platform]y Adobes. Además, [!DNL Profile] se utiliza para alimentar el aprendizaje automático y Sensei. [!DNL Profile] Las API también pueden utilizarse directamente para enriquecer la funcionalidad de soluciones de terceros, sistemas CRM y soluciones propietarias.

| Función | Descripción |
| -----------| ---------- |
| [!DNL Real-time customer profile] | [!DNL Profile] ingiere datos mediante entradas de datos, los combina con datos existentes en el almacén de perfiles y crea o modifica la vista unificada en tiempo casi real. Estos datos, que evolucionan a tiempo con las experiencias de sus clientes, se pueden utilizar para realizar recomendaciones personalizadas. |
| Vista de unión | Una de las características clave de [!DNL Profile] es la capacidad de unificar datos de varios canales. [!DNL Profile] puede proporcionar una vista combinada de los individuos en su base de consumidores, entre conjuntos de datos e identidades vinculadas, conocida como la vista de unión. |
| Edge | Edge Service sirve como marco para la recopilación de datos de baja latencia, la informática conectable y la rápida activación de datos en todos los canales direccionables. Edge Service facilita la personalización en tiempo real. |

**Problemas conocidos**

Ninguna.

Para obtener documentación sobre el producto, consulte Información general sobre [Perfiles](../../profile/home.md).

## Servicio de identidad {#identity}

El servicio de identidad resuelve el desafío fundamental que plantean las identidades desconectadas de sus clientes, debido a la naturaleza fragmentada de sus datos en cada uno de los sistemas dispares en juego en el ecosistema de su organización. Esto se logra mediante el enlace de identidades entre dispositivos y entre los distintos sistemas utilizados por los consumidores para interactuar con su marca. De este modo, el servicio de identidad facilita una comprensión completa de sus clientes y de su comportamiento para que pueda ofrecer experiencias digitales impactantes en tiempo real.

| Función | Descripción |
| -----------| ---------- |
| Gráfico de identidad | Un gráfico de identidad es un mapa de las relaciones entre identidades que se actualiza con la actividad del cliente en tiempo casi real. [!DNL Identity Service] administra un gráfico de identidad visible únicamente por su organización y creado en función de sus datos, denominado gráfico privado. [!DNL Identity Service] aumenta el gráfico privado cuando un registro de datos ingestados contiene más de una identidad, agregando relaciones entre estas identidades. |

**Problemas conocidos**

Ninguna.

Para obtener documentación sobre el producto, consulte [Identity Service Overview](../../identity-service/home.md).

## [!DNL Segmentation Service] {#segmentation}

[!DNL Segmentation Service] define un subconjunto concreto de perfiles de la tienda de perfiles, describiendo los criterios que distinguen a un grupo comercializable de personas de la tienda de perfiles. Los segmentos pueden basarse en datos de registros (como información demográfica) o en eventos de series temporales que representen los puntos de contacto del cliente con su marca.

Por ejemplo: en una campaña de correo electrónico llamada &quot;¿Olvidó comprar sus zapatillas?&quot;, puede que desee un segmento de audiencia de todos los usuarios que buscaron zapatos deportivos en los últimos 30 días, pero que no completaron una compra. O bien, es posible que desee destinatario el contenido web para que solo se muestre a los usuarios con una determinada clasificación en el programa de recompensas.

| Función | Descripción |
| -----------| ---------- |
| API de segmentación | Los servicios RESTful le proporcionan acceso mediante programación para generar segmentos de clientes. |
| Generador de segmentos | El generador de segmentos es un espacio de trabajo sólido e intuitivo para crear segmentos de clientes a partir de sus [!DNL Profile] datos. |

**Problemas conocidos**

Ninguna.

Para obtener documentación sobre el producto, consulte la información general sobre el servicio [de segmentación](../../segmentation/home.md)

## [!DNL Data Governance] {#governance}

Adobe Experience Platform permite a las compañías reunir datos de varios sistemas empresariales para permitir que los especialistas en marketing identifiquen, comprendan y capten mejor a los clientes. [!DNL Experience Platform] incluye una infraestructura de administración de datos end-to-end para garantizar el uso adecuado de los datos dentro [!DNL Platform] y cuando se comparten entre sistemas.

La administración de datos es una serie de estrategias y tecnologías utilizadas para administrar los datos de los clientes y garantizar el cumplimiento de las regulaciones, restricciones y políticas aplicables al uso de los datos. Desempeña un papel clave en [!DNL Experience Platform] varios niveles, incluyendo catalogación, linaje de datos, etiquetado de uso de datos, políticas de acceso a datos e controles de acceso en datos para acciones de mercadotecnia.

Para comenzar con la administración de datos, es necesario comprender a fondo las regulaciones, las obligaciones contractuales y las políticas corporativas que se aplican a los datos del cliente. Desde allí, los datos pueden clasificarse aplicando las etiquetas de uso de datos correspondientes, y su uso puede controlarse mediante la definición de políticas de uso de datos.

El [!DNL Data Governance] marco simplifica y racionaliza el proceso de categorización de datos y creación de políticas de uso de datos a través de la interfaz de [!DNL Experience Platform] usuario y la [!DNL Policy Service] API.

| Función | Descripción |
| -----------| ---------- |
| Entidades de etiquetas [!DNL Experience Platform] | Aplique y administre etiquetas de uso de datos en el nivel de campo de conexión, conjunto de datos y conjunto de datos en base a etiquetas clasificadas predefinidas. |
| Crear directivas de uso de datos | Defina y administre políticas de uso de datos en función de la presencia de etiquetas y acciones de mercadotecnia solicitadas. |
| Comprobar infracciones de directiva | Evaluar si se infringen las directivas de uso definidas al realizar acciones de marketing en [!DNL Platform] datos. |

**Problemas conocidos**

* Ninguna.

Para obtener más información, consulte la información general [sobre el Gobierno](../../data-governance/home.md)de datos).
