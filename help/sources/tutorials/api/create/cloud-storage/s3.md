# Cree un [!DNL Amazon] Conexión base S3 utilizando la [!DNL Flow Service] API

Una conexión base representa la conexión autenticada entre un origen y Adobe Experience Platform.

Este tutorial le guía por los pasos para crear una conexión base para [!DNL Amazon S3] (en lo sucesivo, &quot;el[!DNL S3]&quot;) usando la variable [[!DNL Flow Service] API](https://www.adobe.io/experience-platform-apis/references/flow-service/).

## Primeros pasos

Esta guía requiere conocer los siguientes componentes de Adobe Experience Platform:

* [Fuentes](../../../../home.md): [!DNL Experience Platform] permite la ingesta de datos de varias fuentes, al mismo tiempo que permite estructurar, etiquetar y mejorar los datos entrantes mediante [!DNL Platform] servicios.
* [Sandboxes](../../../../../sandboxes/home.md): [!DNL Experience Platform] proporciona entornos limitados virtuales que dividen un solo [!DNL Platform] en entornos virtuales independientes para ayudar a desarrollar y desarrollar aplicaciones de experiencia digital.

Las secciones siguientes proporcionan información adicional que debe conocer para conectarse correctamente a un [!DNL S3] almacenamiento mediante [!DNL Flow Service] API.

### Recopilar las credenciales necesarias

Para [!DNL Flow Service] para conectarse con su [!DNL S3] , debe proporcionar valores para las siguientes propiedades de conexión:

| Credencial | Descripción |
| ---------- | ----------- |
| `s3AccessKey` | El ID de clave de acceso para su [!DNL S3] cubo. |
| `s3SecretKey` | El ID de clave secreta de su [!DNL S3] cubo. |
| `serviceUrl` | (Opcional) El [!DNL S3] punto final al que conectarse. Este campo es obligatorio cuando [!DNL S3] bucket es específico de la región. El formato de `serviceUrl` es: `https://s3.{REGION}.amazonaws.com/)`. |
| `bucketName` | La variable [!DNL S3] contiene los datos y sus metadatos descriptivos correspondientes. Su [!DNL S3] el nombre del contenedor debe tener entre tres y 63 caracteres de longitud y debe comenzar y terminar con una letra o un número. El nombre del contenedor solo puede tener letras minúsculas, números o guiones (`-`) y no se pueden formatear como direcciones IP. |
| `folderPath` | La ruta a la carpeta de su [!DNL S3] espacio en el que se almacenan los datos. Esta credencial es necesaria cuando el usuario tiene acceso restringido. |
| `connectionSpec.id` | La especificación de conexión devuelve las propiedades del conector de un origen, incluidas las especificaciones de autenticación relacionadas con la creación de las conexiones base y de origen. El ID de especificación de conexión para [!DNL S3] es: `ecadc60c-7455-4d87-84dc-2a0e293d997b`. |

Para obtener más información sobre cómo empezar, visite [this [!DNL Amazon Web Services] documento](https://aws.amazon.com/blogs/security/wheres-my-secret-access-key/).

### Uso de las API de plataforma

## Creación de una conexión base

Una conexión base retiene información entre la fuente y la plataforma, incluidas las credenciales de autenticación de la fuente, el estado actual de la conexión y el ID de conexión base único. El ID de conexión base le permite explorar y navegar archivos desde el origen e identificar los elementos específicos que desea introducir, incluida la información sobre sus tipos de datos y formatos.

Para crear un ID de conexión base, realice una solicitud de POST al `/connections` al proporcionar su [!DNL S3] credenciales de autenticación como parte de los parámetros de solicitud.

**Formato de API**

```http
POST /connections
```

**Solicitud**

La siguiente solicitud crea una conexión base para [!DNL S3]:

```shell
curl -X POST \
    'https://platform.adobe.io/data/foundation/flowservice/connections' \
    -H 'Authorization: Bearer {ACCESS_TOKEN}' \
    -H 'x-api-key: {API_KEY}' \
    -H 'x-gw-ims-org-id: {IMS_ORG}' \
    -H 'x-sandbox-name: {SANDBOX_NAME}' \
    -H 'Content-Type: application/json' \
    -d '{
        "name": "S3 Connection",
        "description": "Connection for an Amazon S3 account",
        "auth": {
            "specName": "Access Key",
            "params": {
                "s3AccessKey": "{S3_ACCESS_KEY}",
                "s3SecretKey": "{S3_SECRET_KEY}"
            }
        },
        "connectionSpec": {
            "id": "ecadc60c-7455-4d87-84dc-2a0e293d997b",
            "version": "1.0"
        }
    }'
```

| Propiedad | Descripción |
| -------- | ----------- |
| `auth.params.s3AccessKey` | La clave de acceso asociada a su [!DNL S3] cubo. |
| `auth.params.s3SecretKey` | La clave secreta asociada a su [!DNL S3] cubo. |
| `connectionSpec.id` | La variable [!DNL S3] id. de especificación de conexión: `ecadc60c-7455-4d87-84dc-2a0e293d997b` |

**Respuesta**

Una respuesta correcta devuelve detalles de la conexión recién creada, incluido su identificador único (`id`). Este ID es necesario para explorar el almacenamiento en el siguiente tutorial.

```json
{
    "id": "4cb0c374-d3bb-4557-b139-5712880adc55",
    "etag": "\"1700d77b-0000-0200-0000-5e3b41a10000\""
}
```
